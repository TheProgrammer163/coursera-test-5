(Watching the video first is both highly reommended and necessary; this is meant to be a resume and a reminder.)

    Note: if you are a beginner, stick to gh-pages only, don't use master one.

  2. Click on Clone/Download, and copy the link to the site

  3. Go in your computer's console

  4. Enter git clone [link that you copied]
    Note: Use ls on a Mac and dir on a PC to see the list of files and the directory

  5. Go to the correct directory of your GitHub file using cd [name of directory]. In my case, I have to enter cd coursera-test

Lesson 3: List of Tools (Lecture 2, Relevent History)

  • www.w3.org/TR/html5 | Site from the W3C for the HTML5 standards.

  • www.caniuse.com | A site that basically keeps track of HTML, CSS, and Javascript standards and more. Note: it also lets you know which browser supports what.

  • validator.w3.org/#validate-by-uri | This is a "Validator", meaning that you can paste the URL of your website link and it tells you if it follows the HTML standards. You can aslo paste your code in it directly or upload your document.

  • www.w3schools.com/browsers/browsers_stats.asp

Lesson 3: Anatomy of an HTML tag

  • Needs to be semantically correct; if <p> is opened first and then <a> is opened second, <a> must be closed before <p>

  • You cannot close a tag at the same time as opening it. If an HTML element is empty, it must have both the opening tag and the closing tag.

Lesson 4: Special HTML Characters (lecture 8)

There are various characters like < that can be interpreted as HTML. If you want to write them, here is a list of different characters.

  • < - &-lt;

  • > - &-gt;

  • Space that keeps 2 words connected: &-nbsp;

  • Ampersand (&) - &-amp;

  • Copyright symbol or © - &-copy;

  • "These double-quotes" - &-quot;
